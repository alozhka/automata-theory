# 1. Краткое описание
## Типы
| Тип данных | Описание                 | Значение по умолчанию | Размер / Кодировка |
| ---------- | ------------------------ | --------------------- | ------------------ |
| dayzint    | Целочисленный тип данных | 0                     | 4 байта            |
| fallout    | Число с плавающей точкой | 0.0                   | 8 байт             |
| strike     | Строковый тип данных     | ""                    | UTF-8              |

## Ключевые особенности
1. При объявлении обязательно указывать тип переменной
2. В логических выражениях значение отличное от 0 (нуля) или "" (пустой строки) интерпретируется, как true
3. Тип переменной после объявления изменить нельзя
4. Неявные преобразования разрешены только в логических выражениях
5. Разрешены явные преобразования, с помощью встоенных(заложенных) в языке функций

## Функции преобразования типов
Для преобразования типов в языке имеются встроенные функции `dayzint(value)`, `fallout(value)`, `strike(value)`

### Преобразование с помощью dayzint(value)
| Принимаемый тип | Описание преобразования                            |
| --------------- | -------------------------------------------------- |
| dayzint         | Возвращается тоже значение                         |
| fallout         | Происходит с откидыванием дробной части            |
| strike          | Строка соответствует целочисленному литералу или литералу с плавающей точкой <br> (в последнем случае дробная часть отбрасывается) <br> разрешается знак "-" перед литералом |

### Преобразование с помощью fallout(value)
| Принимаемый тип | Описание преобразования                       |
| --------------- | --------------------------------------------- |
| dayzint         | К значению добавляется дробная часть равная 0 |
| fallout         | Возвращается тоже значение                    |
| strike          | Строка соответствует целочисленному литералу или литералу с плавающей точкой <br> (в первом случае к целочисленному литералу дробная часть равная 0) <br> Разрешается знак "-" перед литералом |

### Преобразование с помощью strike(value)
| Принимаемый тип | Описание преобразования    |
| --------------- | -------------------------  |
| dayzint         | Представляет в виде строки |
| fallout         | Представляет в виде строки |
| strike          | Возвращается тоже значение |

* Невозможность преобразования приводит к ошибке

## Функции работы со строками

| Сигнатура           | Описание                |
| --------------------| ----------------------- |
| length(str)         | Возвращает длину строки |
| char_at(str, index) | Извлекает отдельный символ из строки по указанной позиции <br> Возвращает строку из одного символа <br> При отсутствующем индексе - ошибка |

# 3. Операторы

## Арифметические операторы:
* При арифметических операциях тип выражения не меняется
* Применять арифметические операции разрешено только для одинаковых типов

| Символы | dayzint                    | fallout                                     | strike       | 
| ------- | ---------------------------| ------------------------------------------- | ------------ |
| `+`     | Сложение                   | Сложение                                    | Конкатенация |
| `-`     | Вычитание или унарный "-"  | Вычитание или унарный "-"                   | Нет          |
| `*`     | умножение                  | Умножение                                   | Нет          |
| `/`     | Деление                    | Деление                                     | Нет          |
| `%`     | Остаток от деления         | Остаток от деления с дробной частью         | Нет          |
| `//`    | Целая часть от деления     | Целая часть от деления с дробной частью = 0 | Нет          |

## Логические операторы: 
* Все типы поддерживают все логические операторы описанные в [expression-grammar](./expression-grammar.md)
* Значение отличное от 0 (нуля) или "" (пустой строки) интерпретируется, как true
* Всегда возвращают dayzint: 0 или 1 

## Операторы сравнения:
* Все типы поддерживают все операторы сравнения описанные в [expression-grammar](./expression-grammar.md)
* Применять операции сравнения разрешено только для одинаковых типов
* Сравнение строк происходит по порядку кодов символов в кодировке
